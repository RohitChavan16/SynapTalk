# Contributing

Thank you for your interest in contributing to SynapTalk. Contributions that improve architecture, reliability, and observability are especially valuable.

## How to contribute

1. Fork the repository and create a feature branch:

```bash
git checkout -b feature/your-topic
```

2. Keep changes focused and split large work into multiple PRs.

3. Include tests and update docs when adding or modifying behavior.

4. Open a PR and include a description, screenshots, and the reasoning behind architecture decisions.

## Review criteria

- Code clarity and modularity
- Tests and reproducible steps
- Architectural fit and operational considerations

We welcome thoughtful engineering contributions that expand the project's systems capabilities.
